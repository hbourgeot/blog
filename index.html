What I can offer?

As you could read above, you would find relevant information on the same topics I named. With the exception that when naming the installation of certain programs in Linux I would not incorporate a sign that is not entered by keyboard, and if you think, yes, I am talking about the "$" sign. Many sites on the internet want to make the user look as if it were that command typed in a terminal, but it is not.

Based on the above, I can go on to affirm that you have seen at least 2 websites that incorporate the commands as follows:

$ sudo apt install (packagename)

This from my point of view, is totally wrong, suppose a customer visits my website and looks for a command to install an application launcher called ULauncher, and this customer is a senior developer with many years of Linux experience and knows how to enter commands, but as he needs to download this program as soon as possible, instead of seeing the command and entering it manually to the terminal, just copy the command.

sudo apt install (packagename)

That would be a good practice, if you are planning to create a website, keep in mind that you should ALWAYS think about the final customer.
